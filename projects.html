I miei Progetti

Timeline 3.0

▼

Progettazione e sviluppo completo di un server multiplayer FiveM basato su framework proprietario realizzato da zero. Il progetto è stato sviluppato utilizzando Lua per la logica lato server client e gameplay, MySQL per la persistenza dei dati e JavaScript, HTML e CSS per la realizzazione delle interfacce utente (NUI).


Sono stati creati numerosi script personalizzati, tra cui sistemi di gestione giocatore, inventario, economia, interazioni, sincronizzazione client-server e salvataggio strutturato delle informazioni. La logica è stata organizzata in modo modulare per facilitare manutenzione, aggiornamenti e ampliamenti futuri del progetto.


Particolare attenzione è stata dedicata all’ottimizzazione delle prestazioni: riduzione del carico lato server, ottimizzazione delle query MySQL e gestione efficiente degli eventi asincroni per garantire stabilità in ambiente multiplayer.


Il progetto ha richiesto attività costante di debugging, gestione dei casi limite e prevenzione di exploit, mantenendo coerenza dei dati e comportamento prevedibile del sistema durante sessioni con più utenti simultanei.

Tecnologie utilizzate: Lua, MySQL, JavaScript, HTML, CSS, TypeScript

Avis WhatsApp Sender

▼

Applicazione desktop sviluppata in Python per l’automazione dell’invio di messaggi WhatsApp tramite API. Il software è dotato di interfaccia grafica realizzata con Tkinter e permette agli operatori di caricare un file Excel contenente numeri di telefono e stato di contatto per inviare comunicazioni in modo massivo ma controllato.


Il sistema legge i dati da fogli .xlsx tramite openpyxl, individua dinamicamente le colonne richieste e filtra i destinatari evitando l’invio ai contatti già segnati come contattati. I messaggi vengono scelti tramite tipologia (prima donazione, triage, studenti) e generati da configurazione JSON esterna per permettere modifiche senza intervento sul codice.


È stata implementata la gestione delle configurazioni tramite file config.json, contenente token di accesso e parametri API WhatsApp Business. L’applicazione costruisce dinamicamente le richieste HTTP verso il servizio di messaggistica utilizzando la libreria requests.


Per il messaggio di triage è presente un sistema di selezione data con validazione del formato, mentre l’applicazione integra controlli di sicurezza come prevenzione invii multipli, gestione delay tra invii e blocco in assenza di dati obbligatori. Sono inoltre gestiti errori di configurazione, formati non validi e assenza di file.


Il progetto include gestione eventi dell’interfaccia, caricamento dinamico delle risorse, parsing strutturato dei dati e separazione tra logica applicativa e configurazione. L’obiettivo è automatizzare comunicazioni organizzative riducendo errori manuali e tempi operativi mantenendo controllo sui destinatari.


Tecnologie utilizzate: Python, Tkinter, openpyxl, requests

Timeline – Discord Bot

▼

Sviluppo di un bot Discord in Node.js dedicato all’integrazione e alla gestione del server community collegato al progetto Timeline. Il bot utilizza la libreria discord.js per gestire eventi, comandi, permessi e interazioni tra utenti e staff all’interno del server.


È stato implementato un sistema modulare di comandi caricati dinamicamente da directory separate, permettendo l’aggiunta di nuove funzionalità senza modificare la struttura principale del codice. Il bot gestisce ruoli, canali temporanei, messaggi embed e pulsanti interattivi tramite componenti UI nativi di Discord.


Ho sviluppato un sistema ticket completo che consente agli utenti di aprire conversazioni private con lo staff tramite interfaccia a pulsanti. Il sistema permette apertura, chiusura e riapertura delle richieste mantenendo ordine e tracciabilità delle comunicazioni.


Il bot integra inoltre funzionalità di monitoraggio del server FiveM tramite API dedicate, permettendo il controllo dello stato del server e l’interazione con i dati di gioco direttamente da Discord.


È presente anche un sistema di monitoraggio Twitch che controlla periodicamente diversi canali e notifica automaticamente la community quando uno streamer va in diretta. La gestione delle richieste avviene tramite chiamate HTTP asincrone con axios e autenticazione tramite token.


Il progetto utilizza MySQL per la gestione persistente dei dati, configurazioni tramite file JSON e variabili d’ambiente, gestione degli errori e riavvio automatico del bot in caso di crash per garantire continuità del servizio.

Tecnologie utilizzate: Javascript, Node.js, MySQL

Roktura – Website

Progetto personale – Sistema sushi in React.js

▼

Progetto personale sviluppato autonomamente con l’obiettivo di applicare in modo pratico le competenze di programmazione, organizzazione del codice e gestione dei dati. L’applicazione è stata progettata seguendo un approccio strutturato: analisi del problema, suddivisione in moduli e implementazione progressiva delle funzionalità principali.


Ho curato la logica applicativa, la gestione degli input utente e la validazione dei dati, assicurando comportamenti prevedibili anche in presenza di errori o valori non validi. L’architettura è stata mantenuta modulare per facilitare modifiche e ampliamenti futuri.


Particolare attenzione è stata dedicata al debugging e alla leggibilità del codice, mantenendo separazione tra configurazioni, logica e presentazione. Sono stati gestiti casi limite, controlli di sicurezza e gestione delle eccezioni per evitare blocchi del programma.


Il progetto è stato utilizzato come esercizio di consolidamento delle competenze di sviluppo: organizzazione dei file, struttura delle funzioni, manutenzione del codice e miglioramento progressivo delle prestazioni attraverso test e correzioni iterative.


Tecnologie utilizzate: React.js, JavaScript, HTML, CSS
